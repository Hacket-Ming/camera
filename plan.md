# 智能摄像头行为识别项目 — 开发计划

## 一、项目目标

通过摄像头采集视频数据，后端实时分析画面，识别"拿东西走"等行为并记录告警。

## 二、技术选型

| 模块       | 技术                          | 说明                                |
| ---------- | ----------------------------- | ----------------------------------- |
| 语言       | Python 3.11+                  | 主力开发语言，AI生态完善            |
| 视频采集   | OpenCV                        | 底层C++，Python绑定性能损失极小     |
| Web 后端   | FastAPI                       | 异步高性能，WebSocket 支持好        |
| 目标检测   | YOLOv8 (ultralytics)          | 检测人体+物体，一行代码加载         |
| 行为识别   | 规则引擎 + 可选动作识别模型   | 先规则，后期可升级为模型            |
| 前端展示   | HTML + JavaScript + WebSocket | 实时查看画面和告警                  |
| 数据库     | SQLite                        | 轻量，记录事件日志                  |
| 配置管理   | YAML                          | 便于修改参数                        |

### 后期性能优化方向（C/C++）

- 用 ONNX Runtime C++ API 替换 Python 推理，提升帧率
- 用 C++ 重写视频采集模块，部署到嵌入式设备（树莓派等）
- 通过 pybind11 封装 C++ 模块供 Python 调用

## 三、系统架构

```
┌─────────────┐     ┌──────────────┐     ┌──────────────┐     ┌───────────┐
│  视频采集层  │────▶│   检测识别层  │────▶│  业务逻辑层   │────▶│  API/展示  │
│  (capture)  │     │  (detector)  │     │  (analyzer)  │     │  (server) │
└─────────────┘     └──────────────┘     └──────────────┘     └───────────┘
  OpenCV             YOLOv8               行为判断/告警         FastAPI
  RTSP拉流           目标检测              事件记录             WebSocket
  帧管理             姿态估计              规则引擎             REST API
```

### 分层职责

1. **capture（采集层）**：负责从不同来源获取视频帧，屏蔽摄像头差异
   - 本地摄像头、RTSP、视频文件统一接口
   - 后期可替换为 C++ 实现

2. **detector（检测层）**：负责对每帧图像做目标检测
   - 人体检测、物体检测
   - 输出检测框+类别+置信度
   - 后期可替换为 ONNX Runtime / TensorRT

3. **analyzer（分析层）**：负责行为判断的业务逻辑
   - 基于检测结果的时序分析
   - 判断行为模式（拿东西、放下、离开等）
   - 生成事件和告警

4. **server（服务层）**：负责对外暴露接口
   - REST API 查询事件
   - WebSocket 推送实时画面
   - 前端页面

5. **common（公共层）**：配置、日志、数据模型等共享模块

## 四、"拿东西走"识别策略

### 阶段一：基于规则（当前实现）

```
检测到人 → 人靠近目标区域 → 目标区域物体数量减少 → 人离开
         ↓
    判定为"拿东西走"行为，记录事件
```

核心逻辑：
- 定义感兴趣区域（ROI）
- 追踪区域内物体数量变化
- 结合人的移动轨迹做关联

### 阶段二：动作识别模型（后期升级）

- 使用 SlowFast / TimeSformer 等视频理解模型
- 训练自定义动作分类器

## 五、IP 摄像头选购策略

| 档位       | 推荐型号                          | 价格     | 特点                      |
| ---------- | --------------------------------- | -------- | ------------------------- |
| 入门验证   | TP-Link Tapo C200 / 小米智能摄像头 | 100-300元 | 便宜，支持RTSP            |
| 项目开发   | 海康威视 DS-IPC-E22H-IW          | 300-800元 | SDK完善，画质好           |
| 专业部署   | 海康/大华 PoE 摄像头              | 800+元   | ONVIF协议，稳定可靠       |

选购要点：
- **必须支持 RTSP 协议**
- 分辨率 1080P 足够
- 需要夜间识别选带红外夜视
- PoE 供电后期部署更方便
